# 学びの地図 基地化計画

## 目的

このサイトを、動画・補足資料・note記事の一覧ではなく、「ハルとおじいさん」の世界を起点に学びを探索できる基地へ発展させる。

中心に置く体験は、答えを受け取ることではなく、問いをたどり、関連する動画・学びの地図・note記事へ進み、自分の現在地と次の一歩が見えること。

## 守る前提

- 既存の黒基調・ネオンカラー・旅/地図/年表の世界観を維持する。
- 既存ページを壊さず、段階的に機能を追加する。
- YouTube、note、各部ページ、進化の年表を同じ地図上のノードとして扱う。
- 新しい見た目を作る前に、まずコンテンツの構造化データを整える。
- 計画変更が必要な場合は、この文書を先に更新してから実装する。

## サイトの強み

- 物語と学習が一体化している。
- 人類史、技術史、言語、数、AI、教育を横断するスケールがある。
- 「答え」ではなく「問い」を中心に設計できる。
- YouTube、学びの地図、note記事という三層の資産がある。
- 進化の年表により、各テーマを文明史の大きな流れに接続できる。

## 現状の課題

- トップページは強いが、まだ「地図」より「目次」に近い。
- ユーザーが自分の現在地を把握しにくい。
- どこから見ればいいか、次に何を見るべきかが明示されていない。
- YouTube、note、各部ページの相互導線が弱い。
- note記事は多いが、テーマ別探索や各部との関係が見えにくい。
- 学習記録、あとで読む、自分の問いといった「自分の地図」機能がない。

## 最終像

トップページを「学びの地図の基地」にする。

ユーザーは次の入口から入れる。

- 動画から入る
- 地図から探す
- 問いから入る
- テーマ別ルートから進む
- 進化の年表から俯瞰する
- note記事で深掘りする

どこから入っても、関連する動画、学びの地図、note記事、問い、次のノードへ進める状態を目指す。

## フェーズ計画

### Phase 1: 構造化データ基盤

目的: すべての改造の土台となる地図データを作る。

実施内容:

- `assets/js/manabimap-data.js` を追加する。
- 部、章、概念、問い、YouTube、note記事、推奨ルートを構造化する。
- 各部に、タグ、関連ノード、関連note、関連動画、代表的な問いを持たせる。
- 既存ページから読み込めるようにする。
- まだ大きなUI変更は行わない。

完了条件:

- ブラウザの `window.ManabiMapData` から地図データを参照できる。
- トップページとnoteページにデータ基盤が読み込まれている。
- 既存の見た目が変わらない。

### Phase 2: トップページに「地図で探す」を追加

目的: トップページを目次から地図へ近づける。

実施内容:

- トップページ上部に「3つの入口」を追加する。
- 「動画で入る」「地図で探す」「問いから入る」を設置する。
- 構造化データを使って、簡易ノードマップを表示する。
- ノードクリックで関連コンテンツの詳細パネルを出す。

完了条件:

- ユーザーがテーマから動画・部・noteへ進める。
- 既存カード群とデザインが自然につながる。

### Phase 3: 各部ページの接続パネル

目的: 各部ページを孤立した補足資料ではなく、地図上の現在地にする。

実施内容:

- 各部ページに「この部の現在地」パネルを追加する。
- 関連動画、関連note、前後のルート、問いを表示する。
- 第8部は第一章・第二章の関係がわかるようにする。

完了条件:

- 各部ページから次の学びへ迷わず進める。
- YouTubeとnoteへの導線が文脈付きになる。

実装方針:

- 共通CSS `assets/css/part-connector.css` と共通JS `assets/js/part-connector.js` で実装する。
- 各部ページには読み込み行のみ追加し、パネル本文は `assets/js/manabimap-data.js` から自動生成する。
- パネルは各ページのカバー直後に表示し、「現在地」「問い」「YouTube」「note」「次に進むなら」を出す。

### Phase 4: note一覧のテーマ別再設計

目的: note記事を単なる記事一覧から、地図の枝道に変える。

実施内容:

- note記事をタグ・部・問いで分類する。
- 「AI」「教育」「言葉と思考」「数字と世界」「知識と検索」「創造性」「学びそのもの」などのフィルターを追加する。
- 各noteカードに対応する部や関連テーマを表示する。
- 空タイトルや準備中カードの扱いを整理する。

完了条件:

- 読みたいテーマからnote記事を探せる。
- note記事が各部の深掘りとして見える。

実装方針:

- `assets/js/note-map.js` で既存noteカードに構造化データを接続する。
- テーマ別フィルターを追加し、AI、教育、言葉と思考、数字と世界、知識と検索、学びそのものから探せるようにする。
- 各noteカードに関連する部へのチップを追加し、記事を地図の枝道として見せる。

### Phase 5: 自分の地図

目的: ユーザーが学びを記録し、自分の現在地を持てるようにする。

実施内容:

- ログインなしで `localStorage` を使う。
- 見た、読んだ、あとで読むを保存する。
- 自分の問いメモを保存する。
- 進捗表示を追加する。
- 「次におすすめ」を表示する。

完了条件:

- ユーザーが自分の学びの進み具合を確認できる。
- サイトが再訪する理由を持つ。

実装方針:

- 共通CSS `assets/css/my-map.css` と共通JS `assets/js/my-map.js` で実装する。
- 保存先は `localStorage` の `manabimap_user_map_v1` とする。
- トップとnoteページに「自分の地図」パネルを表示し、各部の進捗と次に進む候補を出す。
- 各部ページの接続パネル内に「読んだ」「あとで読む」ボタンを追加する。
- noteページでは記事カードを開いた時点で既読として記録し、既読カードを視覚的に表示する。

## 推奨ルート

進化の年表 -> 第1部 -> 第2部 -> 第3部 -> 第8部 第一章

### AIを知りたい人向け

第3部 -> 第8部 第一章 -> 第8部 第二章 -> AI関連note

### 教育を考えたい人向け

第8部 第二章 -> 第3部の教育パート -> 教育関連note

### 言葉と思考を知りたい人向け

第5部 -> 第4部 -> 第8部 第一章

### 数と社会を知りたい人向け

第6部 -> 数字関連note -> 第8部 第二章

### 人類史から入りたい人向け

進化の年表 -> 第1部 -> 第4部 -> 第6部 -> 第8部

## 実装メモ

- データはまず静的なJavaScriptで持つ。
- 後からJSON化できる形にしておく。
- 既存のHTML直書き構造は当面維持する。
- 大きな画面変更はPhase 2以降に行う。
- デザインは既存のネオンカラーとカード表現に合わせる。
